## 1. 問題描述

Bloom Kitchen（`recipe.bloss0m.com`）在 PRD-003 已完成品牌、導覽、首頁動線與食譜瀏覽 UX；PRD-004 補齊 100 篇食譜、早午餐專區與 WEBP 封面。站點**功能與內容已齊**，但**視覺質感**仍偏「規格有、實作未收斂」：

### 1.1 整體質感未達 guideline 意圖

- `guideline/ui/ui-guideline.md` 已定義語意色票、字型階層、便條紙／砧板木等表面，但 `global.css` 與元件樣式**未系統性套用** elevation、字級 token、focus 與按鈕／chip 一致樣式。
- 標題字型仍偏 system-ui 預設，缺少「小廚房雜誌感」的**可讀又溫暖**的排版節奏（行高、字重、區塊留白）。
- 卡片、按鈕、表單在各頁**視覺權重不一**，全站第一印象仍像「可用的靜態站」而非「精心編排的料理內容站」。

### 1.2 首頁與主題專區缺乏「入口儀式感」

- 首頁 hero、搜尋面板、主題專區三卡（早午餐／牛肉／義大利麵）結構正確，但**視覺層級平淡**：專區卡與一般 `note-card` 同質，無法在 3 秒內傳達「這是精選入口」。
- 主題專區頁（`/brunch/`、`/beef/`、`/pasta/`）內容分組完整，但 **hero／分類標題／食譜網格** 節奏相近，專區個性（早午餐 vs 牛肉 vs 義麵）未在版面與裝飾上區分。
- 100 篇 WEBP 封面已上線，首頁與專區卻**未善用封面比例與網格節奏** 提升質感（列表頁 RecipeCard 已有圖，專區首屏仍偏文字牆）。

### 1.3 與 PRD-003 的關係

PRD-003 §6 排除「大規模 UI 視覺重設」。本 PRD **不推翻** ui-guideline 的色系與冰箱便條定位，而是：

- **收斂實作**（token → CSS → 共用元件）
- **升級關鍵入口頁**（首頁 + 三個主題專區）
- **全站基礎質感**（按鈕、chip、卡片、section 節奏）透過 spec-015 一併統一，避免只改首頁造成割裂

## 2. 證據

- `guideline/ui/ui-guideline.md` v1.1 已定義完整色彩／字型／間距，但實作檔 `src/styles/global.css` 未完整對應 display 字級、陰影 token、專區 hero 變體。
- 首頁 `src/pages/index.astro`：hero + `topic-hub-grid` 三卡皆為相同 `note-card card-body`，無專區預覽圖或視覺差異。
- 專區頁 `src/pages/brunch|beef|pasta/index.astro` 共用 `recipe-hero` + 多段 `section`，結構一致、裝飾不足。
- PRD-003 spec-013 已強化 RecipeCard **資訊層級**（時間／設備），本 PRD 不重做資訊架構，僅在共用元件層做**外觀 polish**（邊框、陰影、圖片比例可調）。
- 競品／使用者期待：料理內容站在 mobile 首屏應有**溫暖、清楚、略帶編輯感**（非 SaaS 漸層、非過度裝飾）。

## 3. 目標使用者與場景

| 角色 | 場景 | 本 PRD 要解決的感受 |
| --- | --- | --- |
| 下班想煮飯的人 | 打開首頁 3 秒內決定「搜食材」或「進專區」 | 首屏有層次、專區卡一眼可辨 |
| 週末想做早午餐的人 | 從首頁或導覽進 `/brunch/` | 專區像「一個主題雜誌章節」，不是一般列表 |
| 從搜尋／社群連結進站的人 | 落地首頁或專區 | 信任感、與 Bloss0m 生態品質一致 |

## 5. 方案選項

| 方案 | 內容 | 優點 | 缺點 | 工時 |
| --- | --- | --- | --- | --- |
| **A. 基礎收斂 + 首頁／專區（推薦）** | spec-015 全站 token／元件；spec-016 首頁 + 三專區 | 質感與入口頁一次到位、依賴清楚 | 需原型或 ui-spec 確認專區差異化程度 | 中 |
| B. 只做首頁與專區，不動 global token | 改動面小 | 易與列表／工具頁割裂 | 低 |
| C. 全站每頁重刷 | 含食譜列表、詳情、工具頁版面 | 最一致 | 超出本輪範圍、回歸風險高 | 高 |

**推薦方案 A**：015 建立「質感底層」，016 在底層上做客製化 hero／專區卡／專區頁節奏。

## 6. 不在範圍內

- 更換技術棧或引入 Tailwind / shadcn / 重型 UI library（維持 Astro + 原生 CSS）
- 推翻 ui-guideline 色系（禁止 SaaS 紫漸層、冷灰科技風）
- CMS、後端、會員、個人化推薦
- 網域／部署遷移
- 新食譜內容批次（屬營運另案）
- 食譜列表 URL 篩選、冰箱工具邏輯、導覽結構（PRD-003 已交付，本 PRD 僅視覺）
- **食譜詳情頁、工具頁、政策頁的完整版面重設**（可受 spec-015 元件樣式連帶改善，不另開專頁 redesign task）
- 自動產圖、插畫素材大量新增（可用既有 WEBP 封面與 CSS 裝飾為主）

## 7. 拆解為 Feature / Spec

**Spec 清單：**

- **spec-015** 視覺系統與共用元件收斂
  - 對齊 `global.css` 與 ui-guideline（字型載入策略、字級／間距 token、陰影、圓角、邊框、focus）
  - 統一 `.button`、`.chip`、`.card`／RecipeCard 外觀基礎（不變更 spec-013 資訊 DOM 語意）
  - Header／Footer 視覺微調（間距、分隔、品牌區層級）
  - ui-guideline 升版並記錄與 PRD-003 的差異

- **spec-016** 首頁與主題專區視覺升級
  - 首頁：hero、搜尋面板、主題專區三卡（可含封面縮圖或專區色帶）、區塊節奏
  - 專區頁：`/brunch/`、`/beef/`、`/pasta/` — hero、分類標題、子區塊、食譜網格視覺（結構可沿用，樣式與裝飾升級）
  - 更新 `docs/ui-specs/site/home.md` 與專區相關 ui-spec（或新增 `docs/ui-specs/site/topic-hub.md`）
  - 可選：`/vif-prototype` 產出首頁 + 一專區靜態 HTML 供 approve 前確認

**拆解理由：**

- 015 先建立全站質感底層，016 專注入口與專區，避免 016 寫一堆 one-off CSS。
- 首頁與三專區使用者路徑一致（從首頁三卡進入），合併為一 spec 利於走查與驗收。
- 列表／詳情頁不單獨開 spec，但 RecipeCard 外觀在 015 統一後全站受益。
