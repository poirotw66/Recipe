## 1. 問題描述

Bloom Kitchen（`recipe.bloss0m.com`）已完成三階段建站：100 篇食譜、完整 SEO 基礎建設、品牌 UX 強化與冰箱剩料工具。但**自然流量仍極低**，尚未進入可持續優化的成長循環。

### 1.1 現況基線（2026-06-03）

| 指標 | 現況 | PRD-001 第 3 個月目標 |
| --- | ---: | ---: |
| 食譜篇數 | 100 | 持續成長中 |
| 靜態頁面（build 輸出） | ~140 | 500+ |
| 食材頁 | 20 | 150+（第 6 月） |
| 情境頁 | 8 | 50+（第 6 月） |
| 主題 Hub | 3 | — |
| 封面圖覆蓋 | 100/100 | — |
| Search Console | 未納入 PRD 實作 | 曝光穩定上升 |
| 站內 Analytics | **未實作** | — |
| AdSense | placeholder | 送審／通過 |

### 1.2 核心瓶頸

1. **無法量測** — 程式碼無 GA4／GTM／Plausible；若未系統性使用 Search Console，無法判斷是「沒曝光」還是「有曝光沒點擊」。
2. **規模不足** — 相較 `Spec.md` 建議的 SEO 驗證規模（300 食譜、100 食材、30 情境），目前頁面量仍偏少，長尾覆蓋有限。
3. **主題權威分散** — 內容同時涵蓋台灣一人份家常菜與 Bloom Kitchen 西式 Brunch／義式專區，對 Google 的垂直訊號不夠集中。
4. **競爭激烈** — 「番茄炒蛋」「蒜香義大利麵」等泛詞難以突圍；需改打**長尾＋情境組合**（租屋、10 分鐘、氣炸鍋、清冰箱）。
5. **引流渠道單一** — 主要依賴自然搜尋；Bloss0m 生態僅 Footer 連結，無主動導流或社群曝光。
6. **變現與信任訊號未完成** — `ads.txt` 仍為 placeholder；預設 OG 圖 `/images/og-default.jpg` 不存在，影響非食譜頁分享預覽。

### 1.3 本 PRD 要解決什麼

不是再堆功能，而是建立**可重複執行的流量成長系統**：

```text
量測 → 診斷 → 內容／SEO 優化 → 擴充規模 → 導流 → 每月複盤
```

## 2. 證據

- PRD-003 god-mode 結案明列：第四階段需新 PRD（內容、AdSense、analytics）。
- `src/components/SeoHead.astro` 預設 `ogImage = "/images/og-default.jpg"`，但 `public/images/og-default.jpg` 不存在。
- `public/ads.txt` 為 PLACEHOLDER；`src/lib/site.ts` 的 `adsenseReady` 在無有效 `PUBLIC_ADSENSE_CLIENT` 時為 false。
- 全站無 `gtag`、GTM、Plausible 等追蹤程式碼（僅文件提及 Search Console）。
- 100 篇食譜中僅 **11 篇** `featured: true`；首頁精選與主軸訊號可再強化。
- 新站 SEO 暖機期通常需 **4～12 週** 才會在 Search Console 看到穩定曝光趨勢；在無量測前，「人數好少」無法區分是收錄問題、排名問題或點擊率問題。

## 3. 策略決策

### 3.1 主軸定位（本 PRD 核心決策）

**第一成長主軸：台灣租屋族一人份快手料理**

| 層級 | 定位 | 說明 |
| --- | --- | --- |
| 品牌 | Bloom Kitchen · 今天煮什麼 | 維持 PRD-003 決策 |
| SEO 主軸 | 一人份 × 10 分鐘 × 冰箱剩料 | 對齊 `Spec.md` 原始垂直 |
| 次要專區 | 氣炸鍋、高蛋白、便當菜 | 以情境頁承接長尾 |
| 保留但降優先 | Brunch／西式專區 | 已有 20 篇與 `/brunch/`，作品牌差異化，**不作第四階段主要產能** |

**理由：** 泛食譜詞競爭太高；本站差異化在「從冰箱出發、一人份、租屋友善」— 與冰箱工具、情境 taxonomy 一致，內鏈結構已具備。

### 3.2 內容優先順序公式

每週新內容／優化依下列順序決策：

```text
1. Search Console「有曝光、CTR < 2%」→ 改 title / description / 首段
2. Search Console「查詢詞有曝光、站內無對應頁」→ 新增長尾食譜
3. 情境頁（一人／10 分鐘／清冰箱）食譜數 < 15 → 補該情境內容
4. 食材頁連結食譜 < 5 → 補食材組合食譜或強化內鏈
5. 其餘：依內容日曆輪替設備（氣炸鍋、電鍋）與蛋白質（雞胸、豆腐）
```

### 3.3 流量目標（從現況出發，修正 PRD-001 時程）

原 PRD-001 第 3 個月目標（3,000～10,000 visits）在僅 100 篇食譜、新站前提下**過於樂觀**。第四階段採**分階段、可驗證**目標：

| 階段 | 時間 | 內容規模 | 流量目標（自然搜尋） | 量測目標 |
| --- | --- | --- | --- | --- |
| **Phase 0** | 第 1～2 週 | 維持 100 篇 | 建立基線 | GSC 驗證、sitemap 提交、analytics 上線 |
| **Phase 1** | 第 1 個月 | 100 篇 + SEO 優化 20 頁 | 收錄 ≥ 80 頁；週曝光 > 0 | 每週 GSC 複盤 |
| **Phase 2** | 第 2～3 個月 | 150 篇食譜 | 月自然點擊 100～300 | 辨識 10+ 有效查詢詞 |
| **Phase 3** | 第 4～6 個月 | 200 篇、食材 35+、情境 12+ | 月自然點擊 500～1,500 | 月活頁面 30+ 有曝光 |
| **Phase 4** | 第 7～12 個月 | 300 篇（對齊 Spec MVP） | 月自然點擊 2,000～5,000 | AdSense 送審／通過 |

> 以上為**方向性目標**，實際以 Search Console 基線建立後第一個月數據校正。若 8 週後收錄 < 50 頁，優先排查索引（sitemap、robots、canonical）而非加內容。

## 4. 預期成果

### 4.1 Phase 0：量測基礎（第 1～2 週）

- Google Search Console 驗證並提交 `sitemap-index.xml`
- 站內 analytics 上線（GA4 或 Plausible，擇一）
- 補齊 `public/images/og-default.jpg`
- 建立**月度流量複盤模板**（見 §10）
- 完成 PRD-003 部署後 Manual Testing Checklist

### 4.2 Phase 1：SEO 衛生與診斷（第 1 個月）

- 優先優化 20 篇高潛力頁（精選 11 篇 + 情境核心頁連結食譜）
- 補齊 3 篇仍為 SVG 佔位的封面 webp
- 新增 `404.astro`（目前無效 slug 導向 `/404` 但頁面不存在）
- 食材／情境列表頁 description 與 H1 對齊主軸關鍵字
- 首頁 `featured` 食譜調整為呼應主軸的組合

### 4.3 Phase 2：內容與 taxonomy 擴充（第 2～4 個月）

- 食譜 100 → **150**（批次 spec-017）
- 食材 20 → **30**（補齊高頻缺口：馬鈴薯、培根、鱈魚、雞翅、麵包等）
- 情境 8 → **12**（新增：電鍋料理、剩飯料理、宵夜料理、無肉料理）
- 主題 Hub 3 → **5**（新增：氣炸鍋、10 分鐘料理 Hub）
- 每篇新食譜必須：唯一 slug、webp 封面、≥2 情境標籤、≥3 食材關聯

### 4.4 Phase 3：規模與變現（第 5～6 個月）

- 食譜 150 → **200**
- AdSense：`ads.txt` 真實條目 + `PUBLIC_ADSENSE_CLIENT` 上線
- Bloss0m 主站／Picker／Render 加入 Bloom Kitchen 入口（需主站 repo 協作）
- 評估是否啟動第二批封面產圖（101～200 對應 `docs/recipe-cover-images-for-gemini.md` 擴充）

### 4.5 可衡量指標（North Star）

| 指標 | 定義 | Phase 2 目標 | Phase 3 目標 |
| --- | --- | ---: | ---: |
| **Indexed pages** | GSC 已索引頁數 | ≥ 120 | ≥ 180 |
| **Weekly impressions** | GSC 週曝光 | 趨勢上升 | ≥ 1,000 |
| **Monthly organic clicks** | GSC 月點擊 | 100～300 | 500～1,500 |
| **Queries with clicks** | 至少 1 次點擊的查詢詞數 | ≥ 10 | ≥ 30 |
| **Fridge tool → recipe CTR** | analytics 事件（若實作） | 建立基線 | 較基線 +20% |
| **Recipe count** | 已發布食譜 | 150 | 200 |

## 5. 方案選項

| 方案 | 優點 | 缺點 | 工時 |
| --- | --- | --- | --- |
| **A. 量測 + 主軸內容 + taxonomy 擴充（推薦）** | 可診斷、可複盤、與現有架構一致 | 需每週營運紀律 | 中高 |
| B. 只狂加食譜、不做量測 | 頁面數成長快 | 不知哪裡有效、可能浪費產能 | 中 |
| C. 只做 SEO 技術優化、不加內容 | 索引品質可能改善 | 新站規模不足時效果有限 | 低中 |
| D. 轉向社群／短影音導流 | 可快速帶量 | 超出靜態站核心能力、需另闢流程 | 高 |

**推薦：方案 A**，依 spec-014～019 分批交付；Phase 0（spec-014）必須先於內容擴充。

## 6. 不在範圍內

第四階段明確不做：

- CMS、後台、會員、收藏、留言、評分
- 後端 API、Cloudflare Workers、D1、R2
- AI 即時推薦、個人化登入
- 付費廣告投放（Google Ads）— 可記錄為第五階段選項
- 網域或託管遷移
- 大規模 UI 重設（僅允許與 SEO／轉換直接相關的微調）
- 將 Brunch 西式內容作為唯一擴充方向（降優先，非禁止）

## 9. 風險與對策

| 風險 | 對策 |
| --- | --- |
| 新站長期無收錄 | 檢查 GSC 覆蓋率報告、robots、sitemap、手動要求索引首頁與精選食譜 |
| 有曝光無點擊 | 優化 title／description；對齊搜尋意圖改首段與 H2 |
| 內容品質不均 | spec-017 強制人工審閱；維持步驟詳盡標準 |
| 產能不足 | 每週 3～5 篇節奏優於月度大爆發；先補 GSC 顯示有需求的詞 |
| 主軸與 Brunch 內容打架 | Brunch 保留於 `/brunch/`，新內容 70% 主軸；不在首頁置頂西式 |
| AdSense 未過 | 維持 placeholder；先以流量與收錄為主，達 150 篇再送審 |
| 隱私法規 | analytics 選型需對齊 privacy policy；Plausible 可作輕量方案 |

## 10. 營運節奏（Monthly Traffic Review）

建議每月第一個週一執行，紀錄於 `docs/ops/monthly-traffic-review-YYYY-MM.md`：

### 10.1 Search Console 檢查項

1. **索引**：已索引 vs 未索引頁數；未索引原因 Top 5
2. **曝光／點擊**：月環比；CTR 變化
3. **查詢詞**：曝光 Top 20、點擊 Top 10、CTR 最低但曝光 > 100 的詞（優化候選）
4. **頁面**：曝光 Top 10 頁面；無曝光之精選／情境頁（內鏈候選）
5. **裝置**：行動 vs 桌機 CTR 差異

### 10.2 站內檢查項

1. `npm test` + `npm run build` 通過
2. 新食譜封面覆蓋率 100%
3. 食材頁平均連結食譜數
4. 冰箱工具頁瀏覽量（analytics）

### 10.3 下月行動產出

- 優化清單（≤10 頁）
- 新食譜主題清單（≤15 篇，附目標關鍵字）
- 是否啟動／延後 AdSense 送審
