## 1. 問題描述

Bloom Kitchen 目前全站為**繁體中文**（`lang="zh-Hant"`），食譜與 UI 皆單一語言。若要觸及英／日／韓使用者與對應的長尾搜尋，需要：

- 可索引的**多語言 URL** 與 **hreflang**，避免 Google 只收錄中文或錯配語言。
- **內容與介面**分離：100 篇食譜全文翻譯工量大，需架構支援**分批上線**。
- **食材／情境／冰箱工具**目前以中文名稱比對；多語後必須以**穩定 id**（現有 ingredient `slug`）串聯，否則篩選與工具會失效。

## 2. 證據

- 單語站僅能服務 `zh_TW` 搜尋意圖；英文／日文／韓文食譜關鍵字無對應落地頁。
- 現有 `src/data/ingredients.json` 已具 **`slug` 作為穩定鍵**（如 `egg`、`tofu`），適合升級為多語 `labels` 而不改食譜關聯鍵。
- Astro 5 內建 **i18n routing**（`prefixDefaultLocale: false`）可保留現有繁中 URL，降低 SEO 斷鏈風險。
- PRD-003 god-mode 報告曾提示第四階段可含 analytics／多語；本 PRD 正式納入多語，**不包含**即時機翻 API 或簡體中文站。

## 3. 目標使用者與場景

| 角色 | 場景 | 期望 |
| --- | --- | --- |
| 海外台灣／華語以外讀者 | 搜尋 "Taiwanese tomato egg" 等 | 英文食譜頁可讀、步驟與份量正確 |
| 日本／韓國在台或旅遊者 | 以母語瀏覽家常料理 | `/ja/`、`/ko/` 入口與列表可理解 |
| 既有繁中使用者 | 書籤舊連結 | `/recipes/...` **不變** |
| 營運者 | 分批翻譯 100 篇 | 先 15 篇試點，再主題批次擴充 |

## 4. 預期成果（分階段）

### Phase A — 平台（spec-017）

- Astro i18n：`zh-TW`（根路徑）、`en`、`ja`、`ko`（`/en/`、`/ja/`、`/ko/` 前綴）。
- UI 字典、Header **語言切換**（同 path 切 locale；無譯文時明確 fallback）。
- Taxonomy 資料模型：`slug` + 各語 `labels`；列表／詳情依 locale 顯示。
- SEO：`html lang`、`og:locale`、**hreflang**（含 `x-default` 指向繁中根 URL）、多語 sitemap 條目。
- 政策／about／contact 等**法律與信任頁**至少提供四語靜態文案（可精簡但不可機翻糊弄 AdSense）。

### Phase B — 內容試點（spec-018）

- **15 篇旗艦食譜**四語完整版（title、description、steps、FAQ…），**slug 四語一致**。
- `recipeId`（與 slug 相同或明確欄位）串聯翻譯；`coverImage`、時間數字、equipment id **跨語共用**。
- `verify-site.mjs`：試點 slug 四語檔案成對、圖片存在、schema 通過。

### Phase C — 規模化（spec-018 延續，營運節奏）

- 其餘食譜依主題批次（早午餐、牛肉、義大利麵專區等）補齊 en／ja／ko。
- 驗證門檻可改為「已翻譯集合內 100% 成對」，不強制一次 400 檔。

### Phase D — 工具（spec-019）

- 冰箱剩料工具：輸入／建議以當前語言 label 呈現，**比對用 ingredient `slug`**。
- `public/scripts/fridge-tool.js` 與食譜資料注入需帶 locale 與 id。

## 5. 方案（已選定）

| 決策 | 選擇 | 理由 |
| --- | --- | --- |
| URL | 子路徑；繁中無前綴 | SEO、單網域、現有連結保留 |
| Slug | 四語同一 slug | hreflang、內鏈、營運對照 |
| 內容存放 | 分語系 content 目錄／collection | 靜態站、分批 build、diff 清楚 |
| Taxonomy | `slug` + `labels[locale]` | 已有 slug；工具與關聯不碎裂 |
| UI | `src/i18n/ui/*.json` + `t()` | 不與食譜 markdown 混雜 |
| 翻譯流程 | **Gemini Flash（latest）** 產稿 + Human review（spec-018 `gemini-translation.md`） | 品質與 SEO；禁用 Google Translate／DeepL 等通用機翻 |
| 簡體 | **不做** | 專案規範 |
| 站內即時翻譯 API | **不做** | 靜態、SEO 穩定 |

## 6. 不在範圍內

- 簡體中文（`zh-CN`）站點或內容
- CMS、後端、使用者投稿翻譯
- 子網域多站（`en.recipe.bloss0m.com`）
- Query string 語言（`?lang=en`）
- 每語言不同 slug 的同一道菜
- 自動覆寫既有繁中食譜（繁中為 source of truth，僅新增他語檔）

## 8. 依賴與影響

- **依賴**：`astro.config.mjs`、`src/content.config.ts`、`src/lib/taxonomy.ts`、`src/components/SeoHead.astro`、sitemap 產生器、導覽（`navigation.ts`）。
- **影響**：
  - 頁面路徑由單層改為 locale-aware（Astro i18n 或 `src/pages/[locale]/` 策略，以 spec-017 為準）。
  - Build 時間隨語系與篇數上升；100×4 為長期目標，非第一個 sprint。
  - Search Console：可觀察各語言前綴表現；需提交更新後 sitemap。
  - AdSense：各語言 policy 頁需可讀；廣告位邏輯不變，仍依 `PUBLIC_ADSENSE_CLIENT`。
- **風險**：半套多語（只有 UI 無食譜）→ 以試點 15 篇 + 列表 fallback 文案緩解。hreflang 錯誤 → verify 腳本檢查成對 URL。
