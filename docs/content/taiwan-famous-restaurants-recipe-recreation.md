> 建立日期：2026-06-03  
> 更新日期：2026-06-03  
> 專區路徑：`/restaurant-replicas/`  
> 用途：為「名店還原專區」整理餐廳與代表菜 backlog，撰寫**盡力還原名店風味**的食譜，不作一人份或快手簡化。

## 專區定位

| 項目 | 說明 |
| --- | --- |
| 專區名稱 | 名店還原專區 |
| URL | `/restaurant-replicas/` |
| 與其他專區差異 | 早午餐／牛肉等專區偏家常快煮；本專區以**還原餐廳代表菜**為目標 |
| 份量 | 依餐廳實際上桌份量撰寫（常見 2～4 人份或全份） |
| 時間 | 不刻意壓在 10 分鐘；長時間燉煮、分次處理如實記錄 |
| 標籤 | 食譜 frontmatter 需含 `tags: ["名店還原"]` |

### 還原原則

1. **味型優先**：醬油色、酒香、麻油香、湯頭層次等，對照公開食譜、媒體報導與餐廳常見做法。
2. **工序不省略**：分次爆香、先煎後燉、冰鎮回軟等關鍵步驟需寫進 steps。
3. **食材如實**：名店使用的特殊食材（樹子、破布子、老菜脯、土魠等）應標註採買建議。
4. **不宣稱官方**：文案用「參考 ○○ 名店常見做法」「還原版」，避免暗示餐廳授權。
5. **與家常食譜分開**：同一道菜若站內已有簡化版（如三杯雞蓋飯），名店還原版另開新 slug，並在 intro 說明差異。

## 食譜 frontmatter 規範

名店還原食譜除一般欄位外，建議加上：

```yaml
tags:
  - "名店還原"
  - "台菜"          # 或 小吃、海鮮 等
restaurantSource:
  restaurant: "富錦樹台菜香檳"
  dishName: "樹子水蓮"
  region: "台北"
```

- `restaurantSource` 為選填 schema 欄位，專區頁與食譜頁可顯示來源餐廳。
- `category` 建議用：`台菜`、`小吃`、`海鮮`、`湯品`、`麵食` 等，與餐廳型態對齊。
- `scenarios` 可留空或選與還原無關的情境；**不要**為了進其他專區而加「10 分鐘料理」等標籤。

## 備註

- 餐廳菜單會變動；以菜色名稱與料理型態為還原主軸。
- 還原食譜為媒體與公開做法整理，**非**餐廳官方食譜。
- 商標與餐廳名稱僅作內容標註，避免暗示商業合作。
